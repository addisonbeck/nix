#+TITLE: Gptel Approval Queue System
#+PROPERTY: header-args:emacs-lisp :tangle gptel-approval-queue-tools.el

This file implements a non-intrusive tool approval system for gptel tools. Instead of blocking on approval requests, tools queue their requests and users review them at their convenience via a dedicated buffer interface.

* Introduction

The approval queue system transforms the current modal approval flow into an asynchronous, queue-based workflow. When gptel tools require approval (shell commands or elisp execution), requests are added to a queue and users are notified via mode line indicator and messages. Users review pending approvals when convenient using a dedicated buffer interface (=C-c C-a=).

** Key Features

- *Non-Blocking*: Tool invocations return immediately, queueing approvals for later review
- *User-Controlled Timing*: Review approvals when convenient, not when requested
- *Security Preserved*: All preview, confirmation, and feedback mechanisms maintained
- *Emacs-Native*: Built entirely on Emacs core functionality, no external dependencies
- *Visible Execution*: Shell command output displayed in real-time

** Architecture Overview

The system consists of:
1. *Data Structures*: Approval request structs and queue state
2. *Queue Management*: Functions for adding, retrieving, and maintaining requests
3. *Notification System*: Mode line indicator and message dispatcher
4. *Buffer Interface*: Dedicated buffer for reviewing and acting on requests (Phase 2)
5. *Execution Layer*: Async executors for shell and elisp (Phase 3)
6. *V2 Tool Definitions*: Queue-integrated versions of existing tools (Phase 4)
